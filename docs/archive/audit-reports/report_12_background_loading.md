# UI 规范审计报告 #12：自适应背景与加载性能优化 (Background & Loading)

本报告分析了项目的背景渲染系统与首屏加载体验。该模块已于 2026-04-17 完成 Design Tokens 品牌化升级，确保了从冷启动到进入游戏的全流程视觉一致性。

---

## 1. 响应式背景系统 (Responsive Background)

*   **单图适配策略**：项目统一使用高分辨率竖屏图 (`bg-mobile-portrait.webp`)。
*   **CSS 几何转换**：通过 `object-fit: cover` 与 `scale(1.2)` (横屏模式) 的组合，解决了背景图在不同长宽比下的“留白”或“非预期切割”问题。
*   **视觉占位 (Fallback)**：
    *   **已同步**：背景组件内置了与 `LoadingScreen` 一致的渐变底色 (`#4c1d95`, `#1e3a8a`)，确保在弱网环境下图片加载前无白屏闪烁。

## 2. 加载体验与品牌化回归 (2026-04-17)

`LoadingScreen` 已根据 2026-04-17 SOP 完成品牌色统一：

*   **品牌色植入 (Brand Tokens)**：
    *   **主标题**: 由 `white` 切换为 `brand-amber (#FFB17A)`。
    *   **进度条/文字**: 由 `white` 切换为 `brand-peach (#FFD5AB)`。
    *   **字重**: 标题锁定为 `font-black` 以增强开屏冲击力。
*   **能效驱动渲染**：
    *   **React 降频**: 状态更新频率从 16ms 降至 150ms，大幅降低了加载期间的 CPU 占用。
    *   **CSS 托管动画**: 进度条位移交由 CSS `transition` 处理，避免了 JavaScript 驱动动画对主线程的阻塞。

## 3. 性能指引

*   **资源预加载**: 利用 Next.js 动态导入预加载 `GameInterface` 核心包。
*   **最短展示时长**: 锁定 `MIN_LOADING_TIME = 900ms`，防止在极速网络下加载页一闪而过导致的视觉不适感（Jank）。
