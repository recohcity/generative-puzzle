# UI 规范审计报告 #05：移动端计分卡片 (MobileScoreLayout.tsx)

本报告分析了在移动端受限空间下，计分信息的极限排版与自适应策略。该组件已于 2026-04-17 完成与全局 Design Tokens 的深度同步与回归。

---

## 1. 核心策略：空间压榨 (Space Compression)

移动端的得分面板必须在极其有限的纵向空间（特别是在手机横屏模式下）内完整呈现复杂的计分公式。

## 2. 横竖屏双模排版

*   **Landscape (手机横屏模式 - 空间拉齐)**：
    *   **移除冗余**：取消了所有外边距、背景填充。
    *   **极致紧凑**：间距调整为 `gap-0`。
    *   **字号拉齐**：标题字号已由原先的 `9px` 恢复至与竖屏一致的 `text-[11px]`，确保跨端视觉权重对齐。
*   **Portrait (手机竖屏模式 - 标准移动端空间)**：
    *   主得分控制在 `20px` 左右，既保证了冲击力，又避免了在窄屏手机上发生文字折行。
    *   使用 `font-medium` 保持轻量质感。

## 3. 视觉优化手段

*   **统一配色方案**：
    *   全面使用 `brand-peach (#FFD5AB)` 作为唯一分数色，摒弃了旧有的橙色/黄色混合方案。
*   **卡片透明化**：
    *   响应 SOP 规范，移除了 `bg-white/[0.02]` 和背景模糊，采用 `bg-transparent border border-white/10`，实现更纯粹的极简视觉。
*   **字重标准化**：
    *   所有数值和标题从 `font-bold` / `font-black` 降级为 `font-medium`。
*   **图标同步**：
    *   移除了移动端特色的 Emoji `🏆`，统一改用 Lucide `Trophy` 图标（`#FFD5AB`），实现与桌面端的 100% 视觉对齐。

### ✅ 颜色对比度与回归 (2026-04-17 已修复)
针对原报告提出的 `25%` 透明度文字可读性低的问题，已在 2026-04-17 回归中将所有数值色锁定为全不透明的 `brand-peach`，辅助文字透明度调整为符合 WCAG 标准的层级。

### ✅ 阴影与阴影剥离 (2026-04-17 已修复)
全量剥离了 `shadow-md` 等老旧阴影样式。
