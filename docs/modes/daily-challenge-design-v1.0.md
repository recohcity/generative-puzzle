# Daily Challenge - 每日挑战模式策划定案稿
## Generative Puzzle v2.0 Mode Design
**版本 v1.0 · 2026-05 · 状态：待玩法评审定案**

---

# 0. 文档目的

本文档用于独立定案每日挑战模式。v2.0 主 PRD 只定义每日挑战的平台边界，本文档负责回答：

- 每日挑战为什么好玩。
- 它和自由模式有什么本质差异。
- 玩家为什么会每天回来。
- 成功、失败、重拼、回顾如何形成完整闭环。
- 哪些内容进入 v2.0 首发，哪些延后。

---

# 1. 玩家承诺

每日挑战给玩家的承诺是：

> 今天，全世界面对同一张拼图和同一个倒计时。你要在限定时间内完成，完成得越快、剩余时间越多，成绩越有分量。

这个承诺包含四个关键词：

- **同题：** 不是随机刷出来的一局，而是当天的共同题目。
- **限日限时：** 今天错过就错过；开局后倒计时立刻开始，时间归零即挑战失败。
- **公平：** 参数、形状、切割和初始散开逻辑一致。
- **可挽回：** 首次失败不直接惩罚离场，允许有代价的第二次。

---

# 2. 与自由模式的差异

| 维度 | 自由模式 | 每日挑战 |
| --- | --- | --- |
| 玩家意图 | 掌控、探索、练习 | 同题竞速、打卡、比较 |
| 参数选择 | 玩家自选 | 系统发布 |
| 随机性 | 每局随机 | 当日固定 |
| 成绩意义 | 个人最佳和自由榜 | 当日限时排名、剩余时间、连续记录 |
| 失败处理 | 可随时重开 | DNF 状态机，一次重拼 |
| 结算重点 | 分数拆解 | 是否限时完成、剩余时间、排名、超越比例 |
| 回访理由 | 试不同配置 | 今日题目更新 |

每日挑战不能只是“自由模式锁参数”。它必须提供自由模式没有的体验：共同题、日榜、状态机、重拼标识和历史回顾。

# 3. 核心循环

## 3.1 日常循环

1. 玩家进入 Lobby。
2. 每日卡片展示今日题已就绪、今日剩余时间、局内限定时间、难度和昨日状态。
3. 玩家进入每日挑战。
4. 系统展示今日题目标、日期标识、不可更改参数和限定完成时间。
5. 拼图散开完成，倒计时立刻开始。
6. 玩家在倒计时内完成，或因倒计时归零/中断失败。
7. 进入每日结算。
8. 结算展示是否限时完成、用时、剩余时间、综合分数、排名、超越比例、首次/重拼标识、连续记录。
9. 玩家回到 Lobby，卡片更新为今日已完成或可重拼。

## 3.2 回访循环

每日挑战的长期循环不是“刷分”，而是：

```text
今日题目 → 完成记录 → 日榜反馈 → 连续记录 → 档案表达 → 明天回来
```

# 4. 状态机

| 状态 | 触发 | Lobby 展示 | 路由行为 |
| --- | --- | --- | --- |
| NOT_STARTED | 当日未开始 | 今日挑战已就绪 | 进入 `/play/daily` |
| PLAYING | 首次进入后未结束 | 不在 Lobby 展示 | 刷新/关闭/倒计时归零判定 DNF |
| FIRST_WIN | 首次完成 | 今日已完成 | 访问 `/play/daily` 重定向结果页 |
| FIRST_DNF | 首次中断 | 尚可重拼，剩余 1 次 | 确认后进入 `?retry=1` |
| RETRY_PLAYING | 重拼进行中 | 不在 Lobby 展示 | 刷新/关闭/倒计时归零判定 RETRY_DNF |
| RETRY_WIN | 重拼完成 | 今日已完成，标记重拼 | 访问 `/play/daily` 重定向结果页 |
| RETRY_DNF | 重拼失败 | 今日挑战已结束 | 只能看结果页 |

## 4.1 DNF 判定

以下场景判定 DNF：

- 倒计时归零时仍未完成。
- 游戏开始后刷新页面。
- 游戏开始后关闭标签页。
- 游戏开始后切后台超过阈值并未恢复。
- 客户端崩溃后未在恢复窗口内继续。

首发必须实现倒计时归零、刷新、关闭 DNF；切后台阈值延后。

# 5. 每日题生成

## 5.1 日期口径

- 使用服务端 UTC+8 自然日。
- 每日题发布时间为当日 00:00:00 UTC+8。
- Lobby 的“今日剩余时间”指向次日 00:00:00 UTC+8；局内倒计时使用当日发布的 `time_limit_ms`。

## 5.2 参数发布

每日题由服务端发布：

| 字段 | 说明 |
| --- | --- |
| `challenge_date` | 日期 |
| `seed` | 当日种子 |
| `seed_version` | 发布版本 |
| `difficulty` | 难度 |
| `cut_type` | 切割类型 |
| `shape_type` | 形状类型 |
| `target_piece_range` | 预期碎片范围 |
| `time_limit_ms` | 当日限定完成时间 |
| `time_limit_tier` | relaxed / standard / pressure / extreme |
| `published_at` | 发布时间 |
| `status` | draft / published / archived |

## 5.3 难度策略

首发 Daily Lite 不做基于全服完成率的自动动态难度，但不应使用过于平直的“周一至周日固定难度表”。每日挑战需要保留可控节奏，同时提供足够的随机惊喜。

推荐采用三层难度生成：

```text
基础节奏池 → 日期种子随机扰动 → 特殊挑战标签
```

### 5.3.1 基础节奏池

基础节奏池只负责控制一周的体感起伏，不直接决定最终难度。

| 日期类型 | 基础难度池 | 目的 |
| --- | --- | --- |
| 工作日起步 | Level 2-4 | 保证回归和低摩擦参与 |
| 工作日中段 | Level 3-5 | 稳定挑战 |
| 周末活跃 | Level 3-6 | 提高话题性和参与深度 |
| 轻松收尾 | Level 2-5 | 防止连续高压 |

### 5.3.2 日期种子随机扰动

每日最终难度由 `challenge_date` 对应的种子确定，在基础池内随机抽取，并允许小概率向上或向下扰动。

| 扰动类型 | 概率 | 效果 | 说明 |
| --- | --- | --- | --- |
| 正常日 | 70% | 使用基础池内难度 | 保持稳定 |
| 惊喜加难 | 15% | 基础难度 +1，最高不超过 Level 7 | 制造挑战话题 |
| 轻松日 | 10% | 基础难度 -1，最低不低于 Level 2 | 给连续参与者喘息 |
| 稀有高压日 | 5% | Level 6-7 | 作为社区事件，不连续出现 |

限制：

- 稀有高压日不得连续出现。
- 任意 7 天内 Level 6-7 不超过 2 天。
- 任意 7 天内至少 2 天为 Level 2-4。
- 首发不开放 Level 8 每日挑战，避免早期流失。

### 5.3.3 局内倒计时难度

每日挑战采用局内倒计时制。每道每日题在发布时同时确定 `time_limit_ms`。玩家一旦获得操作权，倒计时立即开始；倒计时归零仍未完成，则本次 attempt 判定 DNF。

倒计时不是装饰 UI，而是每日挑战区别于自由模式的核心压力来源。

#### 时间限制生成

时间限制由当日难度、碎片数量、切割类型、形状类型共同决定：

```text
baseLimit = actualPieces × avgSecondsPerPiece(level)
timeLimit = baseLimit × cutTypeFactor × shapeFactor × pressureFactor
```

推荐首发参数：

| 难度 | 平均每片时间 |
| --- | --- |
| Level 2-3 | 10-14 秒 |
| Level 4-5 | 14-20 秒 |
| Level 6 | 20-28 秒 |
| Level 7 | 28-36 秒 |

| 倒计时档位 | `pressureFactor` | 体验 |
| --- | --- | --- |
| relaxed | 1.35 | 轻松可完成 |
| standard | 1.10 | 稳定挑战 |
| pressure | 0.90 | 明显紧张 |
| extreme | 0.75 | 稀有高压日 |

限制：

- `extreme` 不得连续出现。
- Level 6-7 默认不搭配 `extreme`，除非明确作为稀有高压日。
- Lobby 必须展示当日限定时间，例如 `限时 04:30`。
- 玩家进入前必须知道今日是 relaxed / standard / pressure / extreme 中哪一档。

#### 倒计时失败

倒计时归零时：

1. 立即锁定拖拽和旋转。
2. 记录 `attempt_status = dnf`。
3. 记录 `failure_reason = timeout`。
4. 若为首次 attempt，进入首次 DNF 结算并允许一次重拼。
5. 若为 retry attempt，进入 RETRY_DNF，今日挑战结束。

#### 成功后的时间价值

玩家在倒计时内完成后，结算展示：

- 限定时间。
- 实际完成用时。
- 剩余时间。
- 剩余时间转化的时间奖励。
- 综合分数。
- 当日排名。

排行榜必须直观展示“谁在限时内完成得更快、剩余时间更多、综合分数更高”。

### 5.3.4 参数组合随机化

难度惊喜不只来自 Level，还来自切割和形状组合。

| 维度 | 策略 |
| --- | --- |
| 难度 Level | 基础池 + 随机扰动 |
| 切割类型 | 日期种子抽取，曲线和斜线提高稀有感 |
| 形状类型 | 日期种子抽取，云朵/锯齿制造识别差异 |
| 碎片数量 | 在难度范围内随机，但必须通过可拼性校验 |
| 倒计时档位 | relaxed / standard / pressure / extreme |

### 5.3.5 延后到 R4 的能力

以下能力仍延后到 R4：

- 基于 7 日全服完成率的自动动态难度。
- 人工运营覆盖。
- 盲拼挑战日。
- 个性化难度推荐。

# 6. 倒计时、得分与排名

## 6.1 倒计时规则

- 计时开始：拼图散开完成后，玩家获得操作权的瞬间。
- 计时结束：最后一块完成吸附并触发完成判定。
- 倒计时归零：立即判定本次 attempt 失败。
- 排行榜计时：使用现实流逝时间，不因暂停或失焦冻结。
- UI 暂停：允许打开暂停层，但倒计时继续流逝。

## 6.2 综合分数

每日挑战可以复用自由模式基础分，但必须加入限时完成维度。

推荐公式方向：

```text
dailyScore = basePuzzleScore + timeRemainingBonus + firstAttemptBonus - retryPenalty
```

| 项 | 说明 |
| --- | --- |
| `basePuzzleScore` | 基于难度、切割、形状、旋转、提示计算 |
| `timeRemainingBonus` | 根据剩余时间比例给奖励 |
| `firstAttemptBonus` | 首次完成获得小额纯净完成奖励 |
| `retryPenalty` | 重拼完成可上榜，但综合分扣除小额比例 |

时间奖励建议：

```text
remainingRatio = remainingMs / timeLimitMs
timeRemainingBonus = round(basePuzzleScore × remainingRatio × 0.35)
```

这让玩家既要完成，也要快；但不会让时间奖励完全压过拼图质量。

## 6.3 排名规则

排名优先级：

1. `score` 高者靠前。
2. 分数相同，`remaining_ms` 多者靠前。
3. 仍相同，`duration_ms` 短者靠前。
4. 仍相同，首次完成优先于重拼完成。
5. 仍相同，提交时间早者靠前。

## 6.4 排行榜展示

每日排行榜必须展示：

| 字段 | 展示 |
| --- | --- |
| 综合分数 | 主排名依据 |
| 完成用时 | 如 `03:42` |
| 剩余时间 | 如 `剩余 00:48` |
| 限定时间 | 如 `限时 04:30` |
| 首次/重拼 | 小标识 |
| 失败人数统计 | 可在榜单摘要显示，不进入排名 |

## 6.5 重拼标识

重拼成绩允许上榜，但必须标记：

- UI 显示：`重拼完成` 或小旗标识。
- 排行榜排序：不直接降权，只作为展示标识。
- 社区语义：首次完成是更纯粹的成绩，重拼是有代价的补救。

# 7. 结算设计

## 7.1 首次完成结算

必须展示：

- 今日日期。
- 限定时间。
- 完成用时。
- 剩余时间。
- 综合分数。
- 当前日榜排名。
- 超越全服百分比。
- 连续挑战状态。
- 返回 Lobby。

## 7.2 重拼完成结算

在首次完成结算基础上增加：

- 重拼标识。
- 首次 DNF 提醒不再强调。
- 连续记录保护说明。

## 7.3 DNF 结算

首次 DNF：

- 语气温和。
- 明确失败原因，如“倒计时结束”或“挑战中断”。
- 若为倒计时结束，展示本局完成进度和已用时间。
- 明确剩余重拼 1 次。
- 给出“再次挑战”和“稍后再来”。

重拼 DNF：

- 显示今日挑战已结束。
- 可查看日榜。
- 不出现羞辱性文案。

# 8. 防作弊与校验

服务端拒收以下成绩：

- `challenge_date` 不匹配。
- `seed_version` 不匹配。
- 参数与每日发布参数不一致。
- `puzzle_fingerprint` 不一致。
- `time_limit_ms` 与每日发布参数不一致。
- `remaining_ms + duration_ms` 与 `time_limit_ms` 偏差异常。
- `duration_ms` 过短或异常。
- attempt 状态不允许再次提交。

首发必须实现参数校验和 attempt 状态校验；fingerprint 可随 PRNG 完成度上线。

# 9. UI 需求

## 9.1 Lobby 卡片

卡片状态：

| 状态 | 主文案 | 辅文案 |
| --- | --- | --- |
| 未开始 | 今日挑战已就绪 | 限时 04:30 · 今日剩余 14:23:05 |
| 首次完成 | 今日已完成 | 排名 #123 · 剩余 00:48 · 超越 68% |
| 首次 DNF | 尚可重拼 | 剩余 1 次 |
| 重拼完成 | 今日已完成 | 重拼成绩已上榜 · 剩余 00:21 |
| 重拼失败 | 今日挑战已结束 | 明天再战 |

## 9.2 局内 HUD

每日挑战局内只显示：

- 日期。
- 大号倒计时器。
- 当前综合分或完成进度。
- 限定时间档位标识，如 `pressure`。
- 提示入口。
- 暂停入口。

不显示参数配置面板。

倒计时 UI 要求：

- 剩余时间充足时保持克制。
- 最后 30 秒进入轻微高压态，但不能遮挡拼图。
- 最后 10 秒可使用音效或边缘脉冲提示。
- 时间归零后立即锁定操作并进入 DNF 流程。

# 12. 可玩性验证

上线前至少验证：

- 玩家是否理解“今日同题”。
- 玩家是否理解“限时完成”以及倒计时归零会失败。
- 各难度的 `time_limit_ms` 是否既紧张又可达。
- 首次 DNF 后是否愿意重拼。
- 重拼标识是否被理解为补救而非羞辱。
- 每日难度是否导致连续多日挫败。
- 日榜中的综合分、完成用时、剩余时间是否足够直观。

# 13. 定案前待决问题

| 问题 | 推荐答案 |
| --- | --- |
| 未登录玩家是否可上日榜 | 可试玩，不上正式榜；虚拟身份可本地保存 |
| 重拼是否影响 streak | 保护不清零，但不增加 |
| 每日挑战是否允许暂停 | 允许 UI 暂停，但局内倒计时继续流逝 |
| 是否显示题目难度预告 | 显示难度、切割类型、倒计时档位和限定时间，降低误入 |
| 是否允许历史挑战 | R4 再做，不进首发 |
