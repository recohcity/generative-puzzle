# Zen Mode - 禅意模式策划定案稿
## Generative Puzzle v2.0 Mode Design
**版本 v1.0 · 2026-05 · 状态：待玩法评审定案**

---

# 0. 文档目的

本文档用于独立定案禅意模式。v2.0 主 PRD 只定义禅意模式的平台边界，本文档负责回答：

- 禅意模式为什么不是自由模式关闭分数。
- 玩家在这里追求什么。
- 它如何形成吸引力、传播力和回访理由。
- 如何避免“无目标导致无聊”。

---

# 1. 玩家承诺

禅意模式给玩家的承诺是：

> 不排名，不催促，不评价。你进入一段安静的拼图时间，完成后留下一张好看的瞬间。

禅意模式不是低难模式，而是目标不同：

- 自由模式追求掌控。
- 禅意模式追求沉浸。
- 自由模式问“我能得多少分？”
- 禅意模式问“我能不能安静地完成它？”

---

# 2. 与自由模式的差异

| 维度 | 自由模式 | 禅意模式 |
| --- | --- | --- |
| 玩家意图 | 探索和刷分 | 放松和沉浸 |
| 参数选择 | 玩家掌控 | 系统温和生成 |
| UI | 全量工具和反馈 | 极简低干扰 |
| 时间 | 计时和奖励 | 不显示时间 |
| 分数 | 核心反馈 | 完全移除 |
| 提示 | 可用，影响分数 | 默认不提供 |
| 完成反馈 | 结算分数 | 视觉/声音完成仪式 |
| 回访理由 | 试配置 | 情绪出口、时长、环境音、主题 |

禅意模式必须改变节奏、目标和感官反馈。只隐藏计时器和排行榜不足以成立。

---

# 3. 核心循环

```text
进入安静场景 → 系统生成温和拼图 → 玩家慢慢拼 → 完成仪式 → 截图/停留 → 回到大厅
```

禅意模式没有失败循环。玩家随时退出也不被视为失败。

---

# 4. 生成规则

## 4.1 参数池

| 参数 | 首发推荐 |
| --- | --- |
| 切割类型 | 曲线为主，少量直线 |
| 形状类型 | 云朵形为主，多边形少量 |
| 难度 | Level 3-4 |
| 碎片数量 | 5-12 片 |
| 旋转 | 90 度倍数 |
| 色彩 | 低饱和暖色、夜色、晨光等主题色 |

## 4.2 难度控制

禅意模式的难度不追求简单，而追求稳定流动感。

避免：

- 碎片过多。
- 颜色过近导致识别困难。
- 磁吸过窄。
- 大量细碎尖角。
- 高频音效或强动画。

推荐单局完成时间：

| 玩家类型 | 目标时长 |
| --- | --- |
| 熟练玩家 | 5-10 分钟 |
| 普通玩家 | 8-15 分钟 |
| 新手玩家 | 10-20 分钟 |

---

# 5. UI 与交互

## 5.1 局内 UI

默认隐藏：

- 分数。
- 计时器。
- 排行榜。
- 参数面板。
- 重开按钮。
- 复杂提示。

保留：

- 右上角极小退出按钮。
- 可选音量入口。
- 必要的磁吸反馈。

## 5.2 退出

退出文案应保持中性：

- `离开`
- `回到大厅`

不使用：

- `放弃`
- `失败`
- `未完成`

---

# 6. 感官设计

## 6.1 视觉

禅意模式应使用主题化视觉，而不是普通游戏背景。

首发主题建议：

| 主题 | 视觉语言 |
| --- | --- |
| 清晨 | 柔暖背景、低对比光晕 |
| 雨声 | 冷灰蓝、轻微雨纹 |
| 极夜 | 深靛蓝、星点粒子，后续解锁 |

## 6.2 声音

声音是禅意模式的核心差异化。

首发音频：

- 清晨环境音。
- 柔和吸附音。
- 完成时低频和弦或呼吸音。

后续解锁：

- 雨声。
- 远雷。
- 夜风。

## 6.3 震动

震动必须轻：

- 吸附：短、柔。
- 完成：渐强但不过度。
- UI 点击：极轻。

---

# 7. 完成仪式

完成后不进入传统分数结算。

流程：

1. 最后一块吸附。
2. 背景音轻微展开。
3. 拼图边缘出现缓慢呼吸光。
4. 粒子或柔光扩散 30 秒。
5. 显示极简完成状态。
6. 30 秒后自动淡出回 Lobby，或玩家手动返回。

完成画面允许截图传播，但不强制弹分享。

可显示的短文案：

- `完成`
- `今日禅意时光 +12 分钟`
- `清晨主题`

# 8. 长期进度

## 8.1 禅意时长

累计禅意时长用于档案表达和温和解锁。

| 时长 | 解锁 |
| --- | --- |
| 10 分钟 | 禅意档案维度开启 |
| 1 小时 | 雨声环境音 |
| 3 小时 | 清晨主题变体 |
| 8 小时 | 夜风环境音 |
| 15 小时 | 极简 Lobby 卡片皮肤 |

## 8.2 沉浸称号

| 条件 | 称号 |
| --- | --- |
| 首次完成 | 初入静境 |
| 累计 1 小时 | 听雨者 |
| 累计 5 小时 | 慢拼者 |
| 累计 15 小时 | 静光收藏家 |

称号只用于档案展示，不影响竞技模式。

# 9. 数据写入

## 9.1 zen_user_stats

记录：

- `total_duration_ms`
- `last_session_at`
- `completed_sessions_count`
- `unlocked_audio_keys`
- `unlocked_theme_keys`

## 9.2 game_sessions

禅意模式可选择写入轻量 session，用于完成率分析：

- `game_mode = ZEN`
- `duration_ms`
- `completed`
- `theme_key`
- `audio_key`
- `difficulty`

不写入：

- `score`
- leaderboard projection

---

# 10. 时长同步

采用本地累加 + 阈值同步：

- 客户端每 30 秒本地累加。
- 主动退出同步。
- `visibilitychange` 同步。
- 累计池达到 5 分钟同步。

服务端限制：

- 单次增量上限。
- 单日异常上限告警。
- 不信任客户端直接提交总时长。

# 11. 首发 MVP

## 必须包含

- `/play/zen` 路由。
- 极简全屏 UI。
- 自动生成温和拼图。
- 无分数、无计时、无榜单。
- 环境音。
- 完成仪式。
- 禅意时长累计。
- Lobby 展示累计时长。

## 延后

- 多主题。
- 多环境音。
- 截图贴纸。
- 高级粒子。
- 长期称号完整系统。

---

# 12. 可玩性验证

上线前必须验证：

- 玩家是否理解这里没有分数。
- 无参数选择是否降低进入摩擦。
- 难度是否仍有“轻微参与感”而不是无聊。
- 完成仪式是否足够有记忆点。
- 环境音是否被接受。
- 平均停留时长是否高于自由低难模式。

健康目标：

| 指标 | 首发目标 |
| --- | --- |
| 平均停留 | 5 分钟以上 |
| 完成率 | 40%-70% |
| 手动退出率 | 不作为负面指标 |
| 二次进入率 | 高于自由低难模式 |

---

# 13. 定案前待决问题

| 问题 | 推荐答案 |
| --- | --- |
| 是否完全无提示 | 首发无提示，但保留“轻触显示轮廓 1 秒”的隐藏可访问性方案 |
| 是否自动播放声音 | 首次进入需尊重浏览器策略，用户交互后播放 |
| 是否允许选择主题 | 首发不允许，后续解锁后可选 |
| 是否记录未完成时长 | 记录时长，但不展示失败 |
| 是否自动下一局 | 不自动，避免破坏完成后的停留感 |
