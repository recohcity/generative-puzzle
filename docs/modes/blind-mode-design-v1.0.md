# Blind Mode - 盲拼模式策划定案稿
## Generative Puzzle v2.0 Mode Design
**版本 v1.0 · 2026-05 · 状态：待玩法评审定案**

---

# 0. 文档目的

本文档用于独立定案盲拼模式。v2.0 主 PRD 只定义盲拼的平台边界，本文档负责回答：

- 盲拼为什么不是自由模式的高难变体。
- 玩家在盲拼中到底挑战什么能力。
- 提示、评级、降级如何避免挫败。
- 哪些规则必须先验证手感。

---

# 1. 玩家承诺

盲拼模式给玩家的承诺是：

> 你只看一眼目标，然后靠记忆、形状和空间推理把它拼回来。

盲拼的核心不是分数，而是技艺感。玩家完成后想表达的是“我真的会拼”，而不是“我刷了更高倍率”。

---

# 2. 与自由模式的差异

| 维度 | 自由模式 | 盲拼模式 |
| --- | --- | --- |
| 玩家意图 | 掌控、试配置、刷分 | 记忆、推理、证明技巧 |
| 目标轮廓 | 持续可见 | 记忆窗口后隐藏 |
| 分数 | 主反馈 | 移除，改评级 |
| 旋转 | 15 度细步长 | 90 度粗步长 |
| 磁吸 | 宽容 | 更精准 |
| 提示 | 扣分 | 影响评级 |
| 失败 | 可重开 | 可降级完成 |
| 长期目标 | 刷最佳分 | S 级覆盖率和段位 |

盲拼必须改变玩家的信息结构：从“对着目标复原”变成“在目标缺席时推理”。如果目标轮廓只是半透明隐藏，模式会和自由模式同质化。

---

# 3. 核心体验

## 3.1 技能构成

盲拼挑战四种能力：

1. **短时空间记忆：** 3 秒内记住整体轮廓和关键边。
2. **局部形状识别：** 通过碎片边缘判断邻接关系。
3. **旋转方向判断：** 在 4 个方向中选择正确方向。
4. **低信息决策：** 在没有目标图的情况下判断下一块。

## 3.2 体验节奏

```text
观察目标 → 轮廓消失 → 探索邻接 → 局部吸附 → 形成结构 → 完成评级
```

盲拼的乐趣来自“看不见但逐渐确定”的过程，而不是单纯提高操作难度。

---

# 4. 进入流程

## 4.1 解锁条件

推荐任一条件开放：

- 完成任意一局自由模式。
- 完成一次每日挑战。
- 登录玩家选择“我了解规则，直接进入”。

该条件只为避免新手误入，不作为长期门槛。

## 4.2 开局仪式

1. 进入盲拼页面。
2. 显示目标轮廓和碎片原位 3 秒。
3. 屏幕出现“记住它”的轻量提示。
4. 目标轮廓 800ms 淡出。
5. 碎片散开。
6. 计时开始。

---

# 5. 核心规则

| 参数 | 推荐值 | 待验证 |
| --- | --- | --- |
| 记忆窗口 | 3 秒 | 可测 3/5 秒 |
| 轮廓淡出 | 800ms | 基本定案 |
| 旋转步长 | 90 度 | 基本定案 |
| 磁吸阈值 | 自由模式 55%-70% | 需手感测试 |
| 免费轮廓闪现 | 1 次 | 基本定案 |
| 计费提示时长 | 3 秒 | 可测 2/3 秒 |
| 推荐难度池 | Level 2-6 | Level 7-8 延后 |

## 5.1 为什么是 90 度旋转

盲拼的挑战应集中在记忆和形状推理，而不是在 24 个角度里盲猜。90 度降低无意义旋转挫败，同时保留方向判断。

## 5.2 为什么要缩小磁吸

目标隐藏后，若磁吸过宽，玩家可以靠拖动试探完成，盲拼会变成“扫雷式碰撞”。缩小磁吸能迫使玩家先判断再移动。

---

# 6. 提示系统

## 6.1 第一层：邻接光晕

当玩家拖起碎片，若附近存在正确邻居，相关边缘出现同色光晕。

特点：

- 免费。
- 不计入提示次数。
- 只提示邻接，不提示绝对位置。
- 用于减少完全无信息的挫败。

## 6.2 第二层：轮廓闪现

点击提示后目标轮廓显示 3 秒。

规则：

- 第一次免费，不计评级提示次数。
- 第二次起计入提示次数。
- 影响 A/B/C 评级。

## 6.3 第三层：换个方式完成

玩家随时可以点击“换个方式完成”。

效果：

- 目标轮廓永久显示。
- 进入可完成状态。
- 盲拼档案记录 D 级。
- 不展示失败文案。

# 7. 评级体系

## 7.1 评级目标

评级要表达技艺，而不是复刻自由模式分数。

| 等级 | 玩家感受 | 条件方向 |
| --- | --- | --- |
| S | 精准大师 | 零计费提示、极速、无多余旋转 |
| A | 稳定高手 | 零计费提示、快速 |
| B | 审慎完成 | 少量提示、良好速度 |
| C | 坚持完成 | 未降级完成 |
| D | 换路完成 | 主动降级后完成 |

## 7.2 速度阈值

盲拼不能直接复用自由模式速度阈值。推荐基于自由模式平均时间乘以盲拼系数：

| 难度 | 盲拼时间系数 |
| --- | --- |
| Level 2-3 | 自由平均 × 1.6 |
| Level 4-5 | 自由平均 × 1.8 |
| Level 6 | 自由平均 × 2.0 |
| Level 7-8 | 暂不进入首发评级 |

具体阈值必须通过实测校准。

## 7.3 多余旋转

多余旋转只作为 S 级门槛，不用于惩罚 A/B/C。

原因：盲拼已经高压，过度惩罚旋转会让玩家不敢探索。

---

# 8. 长期进度

## 8.1 S 级覆盖率

盲拼长期目标是 S 级覆盖率。

推荐组合：

```text
3 种切割类型 × 6 个首发难度 = 18 组合
```

v1.1 中的 24 组合包含 Level 7-8，建议延后。首发先用 18 组合，待数据健康后扩展到 24。

## 8.2 段位称号

| 条件 | 称号 |
| --- | --- |
| 首次完成 C 以上 | 盲拼入门 |
| 任意 3 个 A | 形状推理者 |
| 任意 1 个 S | 暗面拼图师 |
| S 覆盖率 50% | 夜航者 |
| S 覆盖率 100% | 无光之眼 |

称号名称可后续调整，但需避免过度中二或压迫感。

# 9. UI 需求

## 9.1 局内 HUD

只显示：

- 正向计时。
- 提示次数。
- 当前评级目标提示。
- “换个方式完成”按钮。
- 极简退出。

不显示：

- 自由模式分数。
- 难度倍率。
- 排行榜按钮。
- 参数面板。

## 9.2 降级按钮

位置：右下角常驻，但视觉低调。  
文案：`换个方式完成`。  
确认弹窗：`显示轮廓并完成这局，盲拼档案将记录为 D 级。`

# 11. 首发 MVP

## 必须包含

- 3 秒记忆窗口。
- 轮廓隐藏。
- 90 度旋转。
- 缩小磁吸。
- 轮廓闪现提示。
- S/A/B/C/D 评级。
- 换个方式完成。
- 盲拼档案投影。

## 延后

- 邻接光晕。
- Level 7-8。
- 盲拼挑战日。
- 复杂段位动画。
- 盲拼专属排行榜。

---

# 12. 可玩性验证

上线前必须验证：

- 3 秒记忆是否足够。
- 90 度旋转是否让难度更合理。
- 磁吸 55% 是否过窄。
- 玩家是否理解“换个方式完成”不是失败。
- C 级完成率是否健康。
- S 级是否稀有但可达。

健康目标：

| 指标 | 首发目标 |
| --- | --- |
| C 以上完成率 | 35%-60% |
| D 级降级率 | 15%-35% |
| S 级率 | 1%-5% |
| 二次进入率 | 高于自由高难模式 |

# 13. 定案前待决问题

| 问题 | 推荐答案 |
| --- | --- |
| 首发是否包含邻接光晕 | 可延后，先验证核心盲拼 |
| S 覆盖率分母是 18 还是 24 | 首发 18，后续扩 24 |
| D 级是否写自由榜 | 默认不写自由榜，只写盲拼档案；如写需弱标记后台来源 |
| 是否允许自由选难度 | 首发允许 Level 2-6，Level 7-8 锁定 |
| 是否设置盲拼日榜 | 不首发，先做档案评级 |
