**版本 v1.1 · 2026-05 · 基准代码 generative-puzzle v1.4.19**

# 1. 架构愿景与设计哲学

游戏将在现有基础之上延伸出四个平行的游戏模式：**自由模式、每日挑战、盲拼模式、禅意模式**。这四个模式的边界并非由简单的"功能增减"来划分，而是由**玩家的心理状态和意图**来界定。

## 设计哲学三原则

- **明确系统与玩家的权力关系：** 自由模式赋予玩家完整掌控权；禅意模式则反转这一关系，由系统决定一切，玩家只负责体验。
- **杜绝模式间的目标污染：** 各模式拥有独立的追求目标，互不干涉。盲拼评级体系与自由模式排行榜完全隔离，避免高分倍率绑架休闲玩家。
- **构建有韧性的挫折化解机制：** 让玩家在失败或卡关时拥有尊严地离场或重试的选项，保证长期留存动机。

## 模式定位总览

| 模式 | 心理状态 | 核心价值 | 主要受众 |
| --- | --- | --- | --- |
| 自由模式 | 掌控 / 探索 | 功能展示台 · 新手着陆区 | 所有玩家 |
| 每日挑战 | 竞争 / 社交 | 日留存 · 全球排名 | 活跃核心玩家 |
| 盲拼模式 | 专注 / 挑战 | 技能天花板 · 几何推理 | 进阶硬核玩家 |
| 禅意模式 | 放松 / 沉浸 | 情绪出口 · 截图传播 | 休闲 / 减压玩家 |

# 2. 模式详述与规则定义

## 2.0 四模式隔离体系设计 (Four-Mode Isolation Architecture)

### 2.0.0 动态准入与用户生命周期 (Access & Privilege Matrix)

为了平衡新手引导与核心玩家的长期留存，系统建立了一套基于"拼图师档案"的阶梯式准入机制。每日挑战不再仅是竞技场，而是开启游戏世界的"仪式钥匙"。

#### A. 身份识别逻辑
| 用户类型 | 识别标志 | 准入策略 |
| --- | --- | --- |
| **纯粹新人 (Newcomer)** | 首次访问（无本地/云端记录） | **首日豁免**：自由模式直接开启，作为功能着陆页。完成首局后触发“入籍仪式”。 |
| **回头游客 (Returning Guest)** | 有本地游玩历史但未登录 | **仪式开启**：开始执行“每日钥匙”链条，保留本地历史最高纪录。 |
| **注册玩家 (Puzzler)** | 已登录账号 | **完整权力矩阵**：根据连续打卡天数（Streak）享受不同等级的豁免特权。 |

#### B. 权力晋升阶梯 (Streak Privileges)
| 连续打卡天数 | 每日解锁链条 (Access Chain) | 核心特权 |
| --- | --- | --- |
| **0 天 (新人)** | 无限制 (Open Access) | 快速上手，感知核心乐趣。 |
| **1 - 6 天** | **每日挑战 (Key)** → 自由模式 → (完成1局) → 盲拼模式 | **禅意模式锁定**。培养“每日一拼”的仪式感。 |
| **7 - 13 天** | **每日挑战 (Key)** → 自由模式 → (完成1局) → **禅意**/盲拼 | **解锁禅意模式**。奖励深度玩家沉浸式放松体验。 |
| **14 天及以上** | **直接开启全模式** (No Key Required) | **一年豁免权**：跳过每日钥匙机制，自由出入。 |

> **豁免权规则：** 达成 14 天连续打卡后，玩家档案将自动获得 `EXEMPT_KEY_PASS` 标识，有效期 365 天。若中途打卡中断，豁免权不回收，但连续打卡计数重置。

> **核心隔离原则：** 四个模式不是"一套代码加几个 if-else 开关"，而是在路由、UI 容器、数据存储和后台管理上的**四个独立实体**。各模式之间必须做到物理与逻辑的彻底隔离，严禁状态与数据相互流窜。

### 2.0.1 前端入口与路由隔离

应用主界面（Lobby）从原有的"直接进入拼图页"重构为**模式大厅**。

- **路由重构：** 剥离原有单一游戏路由，变更为 `/play/free`、`/play/daily`、`/play/blind`、`/play/zen`，每条路由页面独立挂载底层业务组件，底层组件内不堆砌跨模式判断逻辑。
- **已完成后的路由保护：** 玩家访问 `/play/daily` 时，若校验到当天**已用完所有挑战与重拼机会（无剩余次数）**，则强制重定向至结算回顾页。若已首次完成但**尚有重拼机会**，结算回顾页应提供“再次挑战”按钮，允许再次进入游戏（携带 URL 参数如 `?retry=1` 或 Context 标识，确保此次成绩被标记为 `⚑ 重拼`，且不覆盖首次成绩）。

**模式卡片信息外显规则：**

- **每日挑战卡片：** 高亮倒计时（"今日挑战还剩 14:23:05"），外露当天完成状态（未完成 / ⚑ 重拼 / ✓ 已完成）。
- **盲拼模式卡片：** 展示玩家当前最高解锁段位及 S 级覆盖率进度（如"S 级：7 / 24 组合"）。
- **自由模式卡片：** 展示上次游玩的配置参数（难度、切割类型），便于快速复盘或继续。
- **禅意模式卡片：** 展示累计禅意时长和当前沉浸称号。

### 2.0.2 UI 架构分发与裁剪隔离

系统通过抽象出的 `ModeController` 动态挂载各模式专有的 UI Shell，而非在底层核心组件中堆砌判断逻辑。

| 模式 | UI Shell 挂载规则 |
| --- | --- |
| 自由模式 | 全量挂载：`DesktopPuzzleSettings`（参数面板）+ `GlobalUtilityButtons` + `LeaderboardPanel` + 全部计分仪表盘元素 |
| 每日挑战 | **物理剥离**参数配置面板（强制不渲染）；顶部新增醒目"每日挑战日期标识"；锁定难度与切割参数；结算独立展示"超越全球 X% 玩家"竞速排名 |
| 盲拼模式 | **物理剥离**参数配置面板；重构顶部仪表盘：仅显示正向计时器（mm:ss）和已用提示次数（0/1 显示），移除分数、难度系数、旋转效率等自由模式指标；右下角常驻"换个方式完成"降级按钮 |
| 禅意模式 | **剥离所有非游戏面板**（强隐藏仪表盘、排行榜按钮、操作按钮）；强制全屏渲染；仅保留右上角一个极小透明退出图标（尺寸 ≤ 24px） |

**结算流转隔离：**

1. **普通结算（自由 / 每日挑战）：** 各自调用独立结算组件，展示对应模式数据，点击"返回"回到大厅。
2. **盲拼正常结算（S/A/B/C 级）：** 展示评级、用时、提示次数，点击"返回"回到大厅。
3. **盲拼降级结算（D 级）：** 切换挂载自由模式结算组件展示分数，右上角微小标记"档案记录：盲拼 D 级"，分数同步写入自由模式排行榜（无负面标记），盲拼档案同步记录 D 级，两条记录并存互不影响。
4. **禅意模式结算：** 完成后停留 30 秒展示粒子扩散效果（供截图），之后自动淡出回主界面，无分数界面。

### 2.0.3 数据模型与 Supabase 存储隔离

彻底废弃使用单一布尔值（如 `is_blind_mode`）区分模式的脆弱做法，将模式定义提升为系统级枚举实体。

- **全局枚举：** `game_mode: 'FREE' | 'DAILY' | 'BLIND' | 'ZEN'`
- **`game_sessions` 表：** 新增强制字段 `game_mode`，同时新增 `challenge_date DATE` 字段用于每日挑战记录（记录对应的挑战日期，支持历史查询与跨日排行榜追溯）。

> **Supabase Schema 核心升级**
>
> ```
> 现有约束：unique(user_id, difficulty)
> 升级为：  unique(user_id, difficulty, game_mode, challenge_date)
>
> 意义：彻底切断不同模式的成绩覆盖污染，并修复每日挑战会被次日成绩覆盖的灾难性漏洞。
> 说明：
> 1. 每日挑战 (DAILY)：写入真实的 challenge_date，保证每天独立记录。
> 2. 自由模式与盲拼模式：challenge_date 强制写入默认值 '1970-01-01'（或 null），确保玩家永远只保留一条最高历史记录。
> 3. ZEN 模式数据：仅写入 game_sessions 统计累计时长，不参与任何 leaderboard 表的聚合与排行计算。
> ```

- **每日挑战防篡改校验：** 前端提交 `DAILY` 模式数据时，`difficulty` 等参数必须与后端当日全局种子参数强制校验，不符直接拒收。

### 2.0.4 后台运营管理端隔离

后台看板 `/scores` 路由下设置四个独立的模式监控视图：

| 视图 | 核心监控指标 | 用途 |
| --- | --- | --- |
| 自由模式 | 切割类型偏好热度图 / 难度分布 / 形状选择比例 | 观察玩家自由探索偏好，指导参数平衡 |
| 每日挑战 | 以自然日为维度的完成率趋势图 / 防流失漏斗（开局→DNF→重拼→完成） | 动态难度调控依据 + 运营手动覆盖接口（含盲拼挑战日设置） |
| 盲拼模式 | S/A/B/C/D 级分布比例 / 降级率（D 级占比）/ 平均用时分布 | 微调磁吸阈值与提示时长的直接数据支撑 |
| 禅意模式 | 平均停留时长 / 活跃时段分布 / 完成率 | 评估情绪缓冲池实际效用，不使用竞技指标 |

## 2.1 自由模式 (Free Mode)

*探索与练习场 · 心理暗示："我在这里掌控一切"*

### 核心定位

游戏的功能展示台、新手着陆区和完全自定义的实验场。玩家在此熟悉三种切割方式的差异、八个难度级别的手感，以及旋转系统和提示系统的使用策略。无任何外部压力或特殊规则限制。

### 进入方式

通过完整的参数配置页进入，玩家可自由选择：

- 难度等级：Level 1（入门，2-4 片）至 Level 8（大师，16-30 片）
- 切割类型：直线 / 斜线 / 曲线（放射状贝塞尔）
- 形状类型：多边形 / 云朵形 / 锯齿形

### 界面与结算

全量显示所有 UI 元素：实时分数、计时器、提示按钮（3 次免费）、难度系数、旋转效率实时反馈。分数按现有 `ScoreCalculator.ts` 公式计算，计入自由模式全球排行榜。

> **分数公式参考（`ScoreCalculator.ts`）**
>
> ```
> finalScore  = max(100, round(subtotal × difficultyMultiplier))
> subtotal    = baseScore + timeBonus + rotationScore + hintScore
> diffMult    = base × cutType × device × shape
>
> baseScore     : Level 1-8 对应 500 - 8000
> timeBonus     : 动态阈值（5-18 秒/片），极速至慢共 6 档
> rotationScore : 完美旋转 +500 分；超出每次 -10 分
> hintScore     : 零提示 +500 分；超出 3 次每次 -25 分
> ```

## 2.2 每日挑战 (Daily Challenge)

## 2.2 每日挑战 (Daily Challenge)

*社交与竞速 · 心理暗示："我要和全世界在同样的条件下比一比"*

### 核心定位

长期留存的主要驱动器。全服每日使用相同日期哈希作为随机种子，生成同一张拼图（同切割类型、同难度、同形状），所有玩家在同等条件下竞争，共享同一张全球排行榜。

### 2.2.1 每日挑战专属状态机与流转（UI 级定义）

玩家在当天内的交互受控于一个严密的防刷状态机。大厅卡片与路由守卫根据玩家当日状态执行不同的 UI 呈现与流转：

| 内部状态 | 大厅卡片外显 UI | 点击卡片 / 访问路由的行为 |
| --- | --- | --- |
| **0. 未开始 (Not Started)** | 高亮展示："新挑战已就绪" / 难度星级 / 剩余倒计时 | 直接进入 `/play/daily` 开启游戏，开始首次挑战计时。 |
| **1. 进行中 (Playing)** | *（玩家此时在局内，不会看到卡片）* | 若中途强制刷新或杀进程，按 **DNF** 处理。 |
| **2. 首次完成 (First Win)** | 绿色高亮："✓ 今日已完成" / 排名超越比例 | **路由拦截**：重定向至 `/play/daily/result` 结算回顾页。 |
| **3. 未完成 (DNF)** | 橙色警示："挑战中断 · 尚可重拼" / 剩余重拼：1次 | 提示"这是最后的机会"，确认后进入 `/play/daily?retry=1`。 |
| **4. 重拼完成 (Retry Win)** | 灰色低调："⚑ 今日已完成(重拼)" / 排名展示 | **路由拦截**：重定向至 `/play/daily/result` 结算回顾页。 |
| **5. 重拼失败 (Retry DNF)** | 红色锁定："✕ 今日挑战已结束" | **路由拦截**：重定向至 `/play/daily/result`，仅允许查阅榜单，**彻底锁死**。 |

### 2.2.2 拼图生成与防篡改验证（核心前置：确定性生成重构）

每日挑战要成立，**“同等条件竞争”**是物理底线。现有代码的 `PuzzleGenerator` 强依赖不可被设种（Seed）的 `Math.random()`，这会导致全球玩家面临的切割线、碎片颜色、散开位置各不相同。

- **伪随机算法重构（核心技术前提）：** 在执行每日挑战开发前，必须重构 `@generative-puzzle/game-core` 中的生成系统。全局引入可被外部干预的 PRNG（伪随机数生成器，如 `mulberry32` 或 `seedrandom`），彻底替换所有切割网络、碎片着色和散开角度计算中的 `Math.random()` 调用。
- **全局同频（发题端）：** `globalSeed = hash(YYYY-MM-DD + serverSalt)`，全服使用服务端 UTC+8 日期。只要输入的 `globalSeed` 相同，不论玩家设备为何，生成的拼图从多边形外廓、内部切割线网格、每一块的渐变色，到初始散开的坐标与旋转角，必须达到 **100% 像素级的一致性**。
- **校验屏障（收题端）：** 前端提交 `DAILY` 模式成绩时，不仅要在视觉和体验上公平，请求体携带的 `difficulty` / `cutType` 参数必须与后端的"今日参数"强制哈希比对。若有篡改（如试图用简单难度伪造分数），直接返回 403 拒收并计入风控。

### 2.2.3 难度动态调控算法（自适应系统）

以 **7 日**为一个观察周期进行动态自适应调整，算法自动计算下一周期的难度基线（Level 2 - Level 8）。

| 7日全服日均完成率 | 触发动作 | 设计心理学目的 |
| --- | --- | --- |
| **> 60%** (过易) | 下一周期难度上限 **+1 级** | 避免头部玩家产生"缺乏挑战"的枯燥感，拉开分数梯队。 |
| **< 25%** (过难) | 下一周期难度上限 **-1 级** | 防止大面积玩家因连续卡关产生习得性无助而导致整体流失。 |
| **25% – 60%** (健康) | 难度上限保持不变 | 维持手感惯性。 |

- **人工干预与盲拼日：** 运营保留强行覆盖日期的权限（后台 Dashboard 操作）。每月可配置 1-2 次「盲拼挑战日」（提前 24 小时大厅预告），当日强制应用盲拼规则，通关者额外获得双档记录（每日榜单成绩 + 盲拼档案解锁）。

### 2.2.4 竞技公平性与防流失保护

#### A. 绝对流逝计时器（Anti-Pause Exploit）
允许游戏内呼出设置菜单或暂停，**但排行榜计时器不停止（现实时间持续流逝）**。这彻底杜绝了"暂停游戏去截图用第三方软件拼图"的作弊行为，保证竞速榜单的绝对公平。

#### B. 成绩重拼标识（"有代价的第二次"）
重拼完成的成绩（状态 4），依然会被写入排行榜并获得全局排名，但会带有醒目的 `⚑ 重拼` 标记。这满足了玩家不留遗憾的心理，但在顶级高玩社区中，无标识的“首发完成”才是实力的硬通货。

#### C. 动态连续打卡保护
连续打卡天数（用于拼图师档案解锁周常边框等）的核心边界规则：

| 触发场景 | 打卡天数 (Streak) 结算结果 |
| --- | --- |
| 状态 2：**首次完成**当天 | 完美履约：连续天数 **+1** |
| 状态 4：仅**重拼完成**当天 | 留存底线：连续天数 **不变**（不增加，但保护不清零） |
| 状态 3/5：当天**彻底 DNF**，次日首次完成 | 惩罚机制：前置连续天数清零，从 **1** 开始重新计算 |
| 当天**完全未登录/未开始** | 惩罚机制：连续天数直接**清零** |

#### D. Rage Quit 关怀推送
若玩家在触发 DNF 时，后台检测到：**（实际耗时 > 基础标准时间 2倍）且（拼接进度 < 50%）**。
代表玩家遭遇了严重的“卡关愤怒”。回到大厅时，卡片气泡不显示“挑战中断”，而是温和推送："今天这幅确实很难，喝口水休息下，重拼记录照样上榜 ↗"，将离开的情绪软着陆，转化为未来的重试动机。

## 2.3 盲拼模式 (Blind Mode)

*极限与天花板 · 心理暗示："我只靠形状的记忆与推理来完成这张拼图"*

### 核心定位

游戏机制的技能天花板。目标轮廓被隐藏，玩家依靠贝塞尔边缘曲率和扇形几何特征进行形状推理与空间记忆。盲拼模式拥有完全独立的成绩体系，不与自由模式排行榜产生任何交叉。

### 2.3.1 进入仪式（3 秒记忆窗口）

点击盲拼模式后，目标形状正常显示 3 秒（供玩家观察并形成空间记忆），随后画面边缘"盲拼模式"标识渐入，目标轮廓在 800ms 内渐隐消失，碎片开始散开动画，计时器启动。

这 3 秒的记忆窗口是盲拼策略层的起点：玩家必须决定"我要记住什么"。

### 2.3.2 核心机制差异

| 机制项 | 自由模式 | 盲拼模式 |
| --- | --- | --- |
| 目标轮廓 | 持续显示（半透明） | 隐藏（仅进入时 3 秒可见） |
| 旋转步长 | 桌面端 15°（24 档） | 强制 90° 步长（4 档） |
| 磁吸阈值 | 基准值 × 1.0 | 基准值 × 0.55（缩小 45%） |
| 顶部仪表盘 | 分数 / 计时 / 难度系数 / 旋转效率 | 仅显示：正向计时器（mm:ss）+ 已用提示次数 |
| 提示系统 | 3 次免费，扣分制 | 1 次免费（3 秒轮廓闪现），第 2 次起影响评级 |
| 排行榜归属 | 自由模式全球榜 | 盲拼独立档案（S/A/B/C/D 评级体系） |
| 分数产出 | 正常 difficultyMultiplier 计分 | 不产生排行榜分数，以评级为最终成绩 |

> **为何强制 90° 旋转步长：** 普通模式桌面端散开角度 0-345°（24 档）。盲拼无参考图时，大量"超出最优旋转"会触发评级惩罚并引发挫败感。90° 步长将挑战限定为"找到正确四个方向之一"，难度合理且公平。

### 2.3.3 成绩评级体系

#### 评级判定树（自上而下，首个满足条件即为该等级）

| 等级 | 判定条件 | 说明 |
| --- | --- | --- |
| S 级 | 提示次数 = 0 且 速度 ≥ 极速 且 多余旋转 = 0 | 三项全满，技艺精湛 |
| A 级 | 提示次数 = 0 且 速度 ≥ 快速 | 零提示快速完成（旋转效率为 S 级专属门槛，此处不作要求） |
| B 级 | 提示次数 ≤ 1 且 速度 ≥ 良好 | 审慎使用一次提示 |
| C 级 | 完成且未触发降级 | 坚持完成，无论用时与提示次数 |
| D 级 | 触发"换个方式完成"降级后完成 | 尊严离场，成绩同步写入自由模式排行榜 |

> **判定说明：** A 级覆盖了"零提示、高速完成但旋转次数未达最优"的场景，体现了"多余旋转 = 0"作为 S 级的额外专属门槛。
> 例：零提示 + 极速 + 多余旋转 3 次 → 满足 A 级条件，判为 A 级。

#### 第二档提示对评级的影响

每使用一次轮廓闪现（第二档提示），提示次数计数 +1，直接影响评级判定树中的提示次数条件（可能导致 A→B 或 B→C）。盲拼模式不维护独立扣分逻辑，**评级即是最终成绩**。第 1 次使用免费，不计入计数。

### 2.3.4 三档容错机制

#### 第一档：接近提示（免费，随时生效）

当玩家拿起一片碎片，且该碎片存在正确邻居处于 2 倍磁吸阈值范围内时，两者相邻边缘同时亮起相同颜色的光晕。仅确认"邻里关系"，不提示正确位置或角度。零成本，保留探索感。

#### 第二档：轮廓闪现（1 次免费，之后影响评级）

点击提示按钮，目标轮廓显示 3 秒后消失。第 1 次免费（不计入提示次数）；第 2 次起每次提示次数 +1，向下影响评级判定。

#### 第三档：动态降级（主动选择，随时可触发）

界面右下角常驻**"换个方式完成"**按钮（语义是选择而非失败，严禁使用"放弃盲拼"等负面措辞）。玩家点击后：

1. 目标轮廓永久显示，进入与自由模式相同的游戏状态。
2. 盲拼状态解除，计时继续。
3. **盲拼档案：** 固定记录为 D 级完成（计入 S 级覆盖率分母）。
4. **自由模式排行榜：** 按正常公式计算分数并写入（无任何负面标记）。两条记录并存，互不影响。

> **降级分数归属总结**
>
> - 盲拼档案       → 记录 D 级完成，计入覆盖率分母
> - 自由模式排行榜 → 正常计分写入，无标记
>
> 设计意图：玩家降级不会"浪费"这局的时间投入，以"换个方式完成"替代"带着遗憾直接退出"，对次日回访率产生正向影响。

## 2.4 禅意模式 (Zen Mode)

*放松与沉浸 · 心理暗示："我放弃了掌控，系统给我一张拼图，我只管拼"*

### 核心定位

纯粹的情绪出口，用于疲惫时的解压与沉浸式体验。与自由模式的核心差异在于：系统与玩家的权力关系发生反转——自由模式里玩家是主人，禅意模式里玩家是客人。

### 2.4.1 进入方式与拼图生成

没有参数选择界面。点击"禅意模式"后直接黑屏淡入（约 1.2 秒）。每次进入系统自动生成一张全新随机拼图（种子完全随机），参数建议范围如下：

| 参数 | 建议值 | 原因 |
| --- | --- | --- |
| 切割类型 | 曲线（贝塞尔） | 边缘最柔和，视觉最舒缓 |
| 难度级别 | Level 3-4（5-12 片） | 适中难度，15-20 分钟内可完成 |
| 形状类型 | 云朵形 | 曲线边缘与氛围一致 |
| 旋转散开 | 90° 倍数（简化） | 减少挫败感，专注沉浸体验 |

拼图散开动画完成后即可开始拼，无任何倒计时或开始提示。

### 2.4.2 结束流转

1. 完成拼图后触发缓慢粒子扩散效果（持续约 30 秒），供玩家截图保存。
2. 30 秒后自动淡出，回到主界面。不自动加载下一局。
3. 玩家可随时再次进入，系统重新生成一张全新随机拼图。
4. 累计游玩时长计入拼图师档案的"禅意时光"维度（心跳写入策略，见下）。

> **禅意时长计算：防抖心跳写入策略（降低 DB 压力）**
>
> - **核心矛盾：** 如果 30 秒直接触发 Supabase Insert/Update，在高并发下会导致极高的数据库 IO 压力。
> - **优化策略（本地防抖 + RPC 按需落盘）：** 客户端内部每 30 秒进行一次本地累加（存入 LocalStorage 或 Context）。
>   仅在以下三种场景调用 Supabase RPC (`increment_zen_duration`)：
>   1. 玩家主动点击右上角“退出”按钮离开模式。
>   2. 触发浏览器 `beforeunload` 或 `visibilitychange` 事件（应用切后台/关闭页面）。
>   3. 本地累加池达到阈值（如 5 分钟），作为极端防丢失的兜底同步。
> - **意义：** 既防止客户端伪造超长时间，又极大降低后端开销，同时保证意外退出时长不丢失。

### 2.4.3 极简 UI 与视听规范

| UI 元素 | 状态 | 说明 |
| --- | --- | --- |
| 分数显示 | 移除 | 无评判，无压力 |
| 计时器 | 移除 | 消除时间感知 |
| 提示按钮 | 移除 | 保留纯粹手感 |
| 配置按钮 | 移除 | 无法中途更改拼图 |
| 退出图标 | 保留（极小） | 右上角，尺寸 ≤ 24px，透明样式 |
| 磁吸反馈 | 保留 | 手感核心体验 |
| 震动反馈 | 保留 | iOS 音效补偿同步保留 |
| 环境音效 | 自动播放 | 不可关闭，侧边栏可调音量；默认"清晨林声" |

第二套环境音效"雨声/远雷"通过禅意时长解锁（见第 3 节跨模式解锁体系）。

# 3. 元游戏架构：拼图师档案 (Puzzler Profile)

不采用数值累积的 RPG 升级模式，而是通过记录玩家在四个模式中的行为，建构其独一无二的"拼图师画像"。档案的目标不是排名，而是给玩家一面镜子：**我是一个什么类型的拼图玩家**。

## 3.1 档案维度

| 维度 | 来源模式 | 记录内容 |
| --- | --- | --- |
| 探索深度 | 自由模式 | 已尝试切割类型（3 种）/ 形状类型（3 种）/ 最高完成难度 / 累计完成拼图数 |
| 日常成就 | 每日挑战 | 连续打卡天数 / 历史最高全球排名 / 本周速度等级分布（极速×N 快速×N …） |
| 盲拼记录 | 盲拼模式 | S 级覆盖率（已达成/24 组合）/ 当前段位称号 / 连续零提示完成局数 |
| 禅意时光 | 禅意模式 | 累计禅意时长（心跳写入）/ 当前沉浸称号 |

**盲拼总组合数：** 3 种切割类型 × 8 个难度级别 = 24 组合。S 级覆盖率 = 已获 S 级组合数 / 24。

## 3.2 跨模式解锁体系（身份表达与特权）

**核心原则：** 游戏功能遵循阶梯式准入（见 2.0.0），解锁内容则涵盖了玩家表达方式（视觉、配色、音效）以及核心访问特权。

| 解锁条件 | 解锁内容 | 类型 |
| --- | --- | --- |
| 每日挑战连续打卡 7 天 | **开启禅意模式** + 专属"周常"边框 | **功能解锁 + 视觉** |
| 每日挑战连续打卡 14 天 | **365 天全局准入豁免权** + "特约拼图师"称号 | **权力特权 + 称号** |
| 盲拼模式首次获得 S 级 | 全模式通用结算粒子特效（金色） | 视觉 · 成就反馈 |
| 禅意模式累计 1 小时 | 第二套环境白噪音（雨声/远雷） | 听觉 · 沉浸体验 |
| 自由模式完成全部 9 种切割×形状组合 | 专属"探索者"档案称号 | 称号 · 身份标识 |
| 盲拼 S 级覆盖率 ≥ 50%（≥ 12/24） | 禅意模式专属「极夜」配色主题 | 终极联动 · 最高成就 |

> **终极联动设计意图**
>
> 盲拼模式（极度专注紧张）的最高成就，在禅意模式（完全放松）中以配色主题的形式绽放。
>
> 「极夜」配色：深靛蓝背景 + 星点粒子 + 冰蓝色边缘光晕，视觉语言与盲拼"黑暗中推理"形成叙事呼应。
>
> 这是整个元游戏里唯一的跨模式奖励，传达的叙事是：只有真正掌握了形状推理的玩家，才能在最终的安静里，以不同的眼光看见同样的拼图。

## 3.3 解锁进度的可见性设计

所有解锁条件在拼图师档案页面完全透明展示，并显示当前进度，例如：

```
盲拼 S 级：7 / 24 组合  →  距"金色粒子特效"还差 1 次
禅意时长：43 分钟       →  距"雨声白噪音"还差 17 分钟
连续打卡：5 天          →  距"周常边框"还差 2 天
```

进度对玩家完全透明，但不在游戏主界面主动推送通知，避免干扰各模式沉浸感。玩家主动进入档案页时，进度以进度条形式呈现。

# 4. 技术实现参考

## 4.1 四模式架构核心改动清单

基于项目现有 Monorepo 物理架构，四模式改动严格遵循边界划分，高度模块化，不影响现有系统正常运行。

| 模块边界 | 文件路径 | 改动类型 | 具体内容 |
| --- | --- | --- | --- |
| 路由与壳架构 | `app/play/` | 结构重构 | 新增 `/free`、`/daily`、`/blind`、`/zen` 专属路由页面，各自按需挂载底层业务组件 |
| 主工程 Constants | `constants/modeConfigs.ts` | 新增 | 统管各模式独有参数（盲拼 `snap: 0.55` / `rotationStep: 90°` / 禅意音量等），废弃写死数值 |
| 核心算法包 (生成) | `packages/game-core/src/utils/puzzle/` | 重构 | **技术底座：** 彻底剥离所有原生 `Math.random()`，引入支持 Seed 的 PRNG 引擎以确保全局 100% 确定性生成 |
| 核心算法包 (计分) | `packages/game-core/src/utils/score/ScoreCalculator.ts` | 扩展 | 根据 `gameMode` 枚举执行独立分数逻辑；盲拼分数隔离，转为评级体系 |
| 主工程渲染层 | `utils/rendering/puzzleDrawing.ts` | 隔离改造 | 仅根据 `gameMode` 和传入 props 决定是否绘制底图，剥离业务耦合 |
| 主工程状态层 | `contexts/GameContext.tsx` | 状态重构 | 移除旧有硬编码，注入全局 `gameMode` 实体，内部逻辑根据当前 Mode 挂载不同行为流 |
| 后端 Supabase | `docs/game-cloud-supabase-mvp.sql` | Schema 升级 | 新增 `game_mode` 枚举与 `challenge_date` 字段；**核心：`unique` 约束必须升级为 `(user_id, difficulty, game_mode, challenge_date)` 防覆盖** |

## 4.2 每日挑战种子生成

在现有随机种子机制基础上，切换为日期哈希生成，全服使用服务端 UTC+8 日期和盐值保证同步。

```javascript
// 伪代码示意
const dailySeed = hash(`${YYYY-MM-DD}:${SERVER_SALT}`);
const puzzle    = generatePuzzle({ seed: dailySeed, ...dailyDifficultyParams });

// challenge_date 字段记录（UTC+8）
await supabase.from('game_sessions').insert({
  game_mode: 'DAILY',
  challenge_date: toUTC8DateString(new Date()),
  ...sessionData
});
```

## 4.3 感官反馈规范（继承自 v1.4.15）

| 场景 | Android (Haptic) | iOS (Audio Fallback) |
| --- | --- | --- |
| 碰撞画布边缘 | 15ms 短震 | 300Hz 三角波碰撞音 |
| 成功吸附（普通） | `[20, 30, 20]` 节奏震动 | 连续高频点射音 |
| 成功吸附（盲拼） | `[30, 20, 30, 20, 50]` 增强节奏 | 升调确认音（盲拼专属） |
| 盲拼完成 | `[30, 20, 30, 20, 50]` 最终震动 | 高频 sine 从低到高扫过 |
| UI 按钮点击 | 10ms 微震 | 交互点击音效 |

盲拼模式下视觉线索减少，音效和震动成为更核心的正确反馈信号，建议适当增强幅度。

# 5. 开发优先级与里程碑

## 5.1 实施路径

| 阶段 | 内容 | 工作量 | 依赖 |
| --- | --- | --- | --- |
| P0 | 路由架构重构 + 每日挑战（日期种子 + 独立排行榜 + `challenge_date` 字段） | 3-4 天 | 现有排行榜系统 |
| P1 | 盲拼模式 MVP（隐藏轮廓 + 90° 旋转 + 磁吸缩小 + 仪表盘改造 + S/A/B/C 评级） | 4-6 天 | P0 并行 |
| P2 | 盲拼容错完善（接近高亮 + 降级流转 + D 级双写） | 2-3 天 | P1 |
| P3 | 禅意模式（极简 UI + 自动生成 + 心跳时长 + 完成粒子） | 2-3 天 | P0 并行 |
| P4 | 元游戏档案页面（四维度记录 + 进度可视化） | 3-5 天 | P1 P3 |
| P5 | 跨模式解锁体系（5 条解锁逻辑 + 视觉/音频资产） | 3-5 天 | P4 |

## 5.2 发布建议

- **首发（P0 + P3）：** 路由重构 + 每日挑战 + 禅意模式。复用现有基础设施最多，用户感知价值高，开发风险低。同时建立四模式路由骨架，为后续阶段奠基。
- **第二版（P1 + P2）：** 盲拼模式完整体验。完成后四个模式全部上线，档案系统可同步集成。
- **持续迭代（P4 + P5）：** 元游戏层逐步丰富，解锁内容可分批上线，无需等待全部完成后统一发布。
